Installation du connecteur :
    Installation de la version :
        Pour le premier passage sur la branche, comme pour une mise à jour normale, faire un "/bin/ose update"
        Lors du choix de version saisir (bien que le choix ne soit pas proposé dans la liste) : FJ_Pegase_Connecteur
        Dans le cas ou l'on est déjà sur la branche, faire un git pull dans le répertoire de OSE pour récupérer les modifications de la branche

        Pensez à installer le driver postgresql qui n'est pas présent dans les versions précédentes de OSE.

    Configuration du fichier config.local.php
        - Paramétrer la base de données DRE et la récupération des étapes
        Pour cela, il faut ajouter dans le fichier config.local.php la config suivante
            'pegase'    => [
                'dre'    => [
                    'driver'   => 'Postgresql',
                    /* IP ou nom DNS du serveur de base de données */
                    'host'            => 'xxx.votre-etablissement.fr',
                    /* Port d'accès au serveur */
                    'port'            => XXXX,
                    /* Nom de la base de données */
                    'dbname'          => 'XXXX',
                    /* Nom d'utilisateur */
                    'username'        => 'XXXX',
                    /* Mot de passe */
                    'password'        => 'XXXX',
                ],
                /* Le type d'étapes a récupèrer (code du type dans pégase), si cet argument est laissé vide, ce sont les éléments les plus hauts de l'arbre qui sont récupéré
                'etapes' => ['XXX','YYYY','FORMATION'],
            ],

         Exemple d'étapes :
            'etapes' => ['FORMATION'],
         ou
            'etapes' => ['ANNEE'],

    Utiliser le connecteur Pégase :
        Pour créer les tables tampons liés à Pégase (PEG_XXXX) :
            "/bin/ose pegase install"
        Pour synchroniser les données de DRE dans les tables tampons
            "/bin/ose pegase synchronisation"

    La version fournie des vues synchronisées qui puise dans les tables tampons est en cours de développement, il faudra les mettre en place pour avoir une remontée des données dans OSE


Mise en place des vues sources :
    Les vue_sources sont disponibles dans le projet dans le dossier suivant : admin/pegase/vue_source
    Ces vues sources sont fournie à titre d'exemple et doivent être personalisé selon votre besoin.
    Pour que les vues sources fonctionne en l'état sans aucune modification il y a quelque prérequis :
        - Renseigner dans la colonne 'source_code' de la table type_intervention le code métier pégase des types d'interventions
        - Ajouter le code pégase des structures dans la colonne 'AUTRE_1' de la table structure pour faire la correspondance

    Information complémentaire :
        - Il est possible de modifier les vue_source pour pointer sur d'autre colonne ou faire d'autre jointure
            --> Par exemple si vous utilisez déjà la colonne AUTRE_1 de structure, vous pouvez changer la vue pour pointer sur AUTRE_2
        - Toute les données remontes dans la vue source, mais ne pourront pas forcément être synchronisé
            --> Exemple : une étape sans structure sera présent dans la vue source, mais ne sera pas possible de remonter dans OSE, un message d'erreur apparaitra lors de la synchronisation sur la page OSE.
